# 运行指南

面向本仓库的**最短路径**：准备数据 → 训练（可选）→ 启动桌面检测系统。

---

## 1. 环境准备

- 安装 **Python 3.10+**（建议 64 位）。
- 打开终端，**进入项目根目录**（与 `train_yolo11.py`、`dataset_visdrone` 同级）。

**训练与命令行推理：**

```bash
pip install ultralytics pyyaml
```

**桌面应用（PyQt6 + 检测界面）：**

```bash
pip install -r detection_app/requirements.txt
```

国内下载慢可使用清华镜像，例如：

```bash
pip install -r detection_app/requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

---

## 2. 数据集

将 **VisDrone 2019-DET** 按仓库目录放好，保证存在：

- `dataset_visdrone/data.yaml`
- `dataset_visdrone/VisDrone2019-DET-train/`（含 `images`、`labels`）
- 验证集、测试集目录与 `data.yaml` 中一致  

无需手改路径：训练脚本会自动生成本机用的 `dataset_visdrone/data_local.yaml`。

---

## 3. 训练模型

在项目根目录执行：

```bash
python train_yolo11.py
```

常用参数示例：

```bash
python train_yolo11.py --model yolo11s --epochs 300 --imgsz 640 --batch 16
python train_yolo11.py --imgsz 1280
python train_yolo11.py --multi_scale
python train_yolo11.py --resume
```

默认结果目录：**`runs/train/yolo11_visdrone/`**，权重在其中的 **`weights/best.pt`**、**`last.pt`**。

验证与单张推理：

```bash
python train_yolo11.py --mode val --weights runs/train/yolo11_visdrone/weights/best.pt
python train_yolo11.py --mode predict --weights runs/train/yolo11_visdrone/weights/best.pt --source 某张图片.jpg
```

---

## 4. 生成指标图（可选）

根据训练产生的 `results.csv` 生成 8 张 PNG（损失、mAP、学习率等）：

```bash
python generate_metrics_charts.py
```

若 `results.csv` 不在默认路径，请指定：

```bash
python generate_metrics_charts.py --csv runs/train/yolo11_visdrone/results.csv --out runs/train/yolo11_visdrone/charts
```

---

## 5. 启动桌面检测系统

```bash
cd detection_app
python main.py
```

- 首次运行会生成 **`config.json`**、**`data/users.json`** 等。  
- **默认登录**：用户名 `admin`，密码 `admin123`（可在登录界面注册新用户）。  

**模型路径：** 程序默认指向 `runs/detect/runs/train/yolo11_visdrone/weights/best.pt`。若你实际训练在 **`runs/train/yolo11_visdrone/`**，请打开应用内 **「模型管理」**，将模型文件选为你的 **`best.pt`**，或编辑 **`detection_app/config.json`** 中的 `model_path`、`metrics_dir`。

---

## 6. 常见问题（Windows / Anaconda）

**报错：`ModuleNotFoundError: No module named 'PyQt6'`**  
→ 执行：`pip install -r detection_app/requirements.txt`

**报错：`DLL load failed` / `找不到指定的程序`（与 QtWidgets 相关）**  
→ 多为 Anaconda 自带 Qt 与 pip 的 PyQt6 冲突。请先按 **`detection_app/requirements.txt`** 安装依赖（其中对 PyQt6 有 **`>=6.4,<6.10`** 约束）；仍失败时，在已激活的 conda 环境中运行 **`detection_app/fix_pyqt_conda.bat`**，或安装 **Microsoft VC++ 2015–2022 x64 可再发行组件**。
